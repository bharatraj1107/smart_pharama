Smart Pharma System – Dev Run Report + External Explanation (Q&A)

1) Project Overview (for external audience)
This is a full-stack MERN application for pharmaceutical packaging companies.
Main goals:
- Manage inventory (Foils and Cylinders)
- Create task workflows and track task status (pending → in-progress → completed)
- Enable QR-based foil tracking and validation
- Track attendance with salary/earnings calculation
- Maintain audit logs for important actions
- Enforce role-based access control (CEO, Admin, Manager, Worker)

Technology stack:
- Frontend: React 18 (react-scripts), React Router
- Backend: Node.js + Express
- Database: MongoDB + Mongoose
- Auth: JWT + bcrypt, with OTP-based signup verification (mock email)
- QR/Barcode: server-side QR generation, client QR scanning (html5-qrcode), barcode generation
- File uploads: Multer (task images, waste images)

4) “Frequently Asked Questions” for external presentation
Q1: What is the purpose of QR in this system?
A: Each foil has a unique QR payload. When a worker starts a task, the scanned QR payload is validated against the inventory in the worker’s company. This reduces mistakes and ensures correct foil consumption/remaining balance updates.

Q2: What happens when a worker completes a task?
A: Worker submits used/waste/remaining KG plus an optional waste image. The backend marks the task as completed and updates inventory by consuming the old foil record and optionally creating a new foil record with a new QR payload for the remaining KG.

Q3: How is attendance salary calculated?
A: The backend calculates hours worked from check-in and check-out. Salary is computed using:
- Per hour: hoursWorked × hourlyRate
- Per day: full day pay if hours ≥ 4, otherwise half day pay
Absent status results in zero earnings.

Q4: How is security handled?
A: JWT is used for authentication. Endpoints are protected by middleware that checks user role and company-scoped access rules.

Q5: Is OTP real email?
A: The current implementation uses a mock transporter. OTP is logged to the server console to simulate email verification.

Q6: What do audit logs store?
A: Audit logs store actions such as create/edit/delete for inventory and tasks, including actor information and before/after state.
